# SBS Program Builder: Progression Scheme Color Guide

<progression_system_overview>
The Stronger By Science program builder utilizes a color-coded cell system to distinguish between different progression methodologies. Each color represents a distinct approach to progressive overload, allowing for customized programming based on exercise type, training goals, and individual response patterns.
</progression_system_overview>

<progression_schemes>
## Color-Coded Progression Categories

### Light Burgundy Cells: "Rep Increase" Progression
<scheme_definition>
**Application**: Primary progression scheme adapted from novice hypertrophy protocols
**Mechanism**: Progressive rep increases within established ranges before load advancement
**Best Use**: Isolation movements, accessory exercises, hypertrophy-focused training
</scheme_definition>

<implementation_protocol>
```
Implementation Steps:
1. Establish baseline rep range (e.g., 8-12 reps)
2. Progress by adding reps across all sets
3. When upper rep threshold achieved, increase load
4. Return to lower rep range with new weight
5. Repeat cycle for continued progression
```
</implementation_protocol>

### Gray Cells: "Set Increase then Rep Increase" Progression
<scheme_definition>
**Application**: Volume-focused progression from novice hypertrophy methodology
**Mechanism**: Systematic set increases followed by rep progression
**Best Use**: Compound accessory movements, volume accumulation phases
</scheme_definition>

<implementation_protocol>
```
Implementation Steps:
1. Begin with baseline set and rep parameters
2. Add sets until upper volume threshold reached
3. Progress to rep increases within set framework
4. Increase load when rep and set targets achieved
5. Reset to baseline volume with new weight
```
</implementation_protocol>

### Teal Cells: "Classic Overload" Progression
<scheme_definition>
**Application**: Traditional strength training progression model
**Mechanism**: Pre-established set/rep targets with load increases upon completion
**Best Use**: Main compound movements, strength-focused training blocks
</scheme_definition>

<implementation_protocol>
```
Implementation Steps:
1. Assign specific set and rep goals
2. Execute training with target parameters
3. Increase load when goals completed successfully
4. Maintain set/rep framework with new weight
5. Continue linear progression until plateau
```
</implementation_protocol>

### Yellow Cells: "Fixed Total Reps" Progression
<scheme_definition>
**Application**: Volume-controlled progression system
**Mechanism**: Predetermined total rep count distributed across variable sets
**Best Use**: Strength endurance training, competition preparation phases
</scheme_definition>

<implementation_protocol>
```
Implementation Steps:
1. Establish total rep target (e.g., 25 total reps)
2. Distribute reps across variable set numbers
3. Progress by completing target reps with fewer sets
4. Increase load when rep efficiency maximized
5. Reset with new weight and rep distribution
```
</implementation_protocol>

### Peach Cells: "Set-by-Set" Progression
<scheme_definition>
**Application**: Advanced autoregulation system with reverse pyramid default
**Mechanism**: Individual set load management with per-set progression tracking
**Best Use**: Main lifts requiring precise load management, advanced trainees
</scheme_definition>

<implementation_protocol>
```
Implementation Steps:
1. Establish base loading for each set position
2. Progress individual sets based on performance
3. Maintain relative intensity relationships
4. Adjust loads per set based on RPE/performance
5. Systematically increase across all set positions
```
</implementation_protocol>
</progression_schemes>

<selection_guidelines>
## Progression Scheme Selection Framework

### Exercise Type Considerations
```
Main Compound Movements:
- Primary: Teal (Classic Overload) or Peach (Set-by-Set)
- Rationale: Precise load management critical for strength development

Accessory Compounds:
- Primary: Gray (Set then Rep Increase) or Light Burgundy (Rep Increase)
- Rationale: Volume flexibility important for hypertrophy stimulus

Isolation Movements:
- Primary: Light Burgundy (Rep Increase)
- Rationale: Simple progression suitable for single-joint exercises
```

### Training Phase Applications
```
Strength Phases:
- Recommended: Teal, Peach
- Focus: Load progression with controlled volume

Hypertrophy Phases:
- Recommended: Light Burgundy, Gray
- Focus: Volume accumulation with systematic progression

Competition Preparation:
- Recommended: Yellow, Peach
- Focus: Skill practice and autoregulation
```
</selection_guidelines>

<advanced_considerations>
## Optimization Strategies

### Progression Scheme Rotation
- **Periodization**: Rotate schemes based on training phase and goals
- **Adaptation**: Change schemes when progress plateaus
- **Individual Response**: Adjust based on personal response patterns

### Performance Monitoring
```
Key Metrics by Scheme:
- Rep Increase: Rep progression rate, load advancement frequency
- Set Increase: Volume tolerance, adaptation timeline
- Classic Overload: Strength progression rate, plateau frequency
- Fixed Reps: Efficiency improvements, set reduction rate
- Set-by-Set: Individual set performance, autoregulation accuracy
```

### Troubleshooting Common Issues
1. **Stalled Progression**: Consider scheme modification or parameter adjustment
2. **Excessive Fatigue**: Reduce volume or change to less demanding scheme
3. **Poor Adaptation**: Evaluate scheme appropriateness for exercise type and individual
</advanced_considerations>
