# Parsons Exercise Best Practices

Parsons problems are a type of code completion problem in which the learner must place blocks of correct, but mixed up, code in the correct order. In this exercise type, learners are presented with jumbled up lines of code and are expected to put them in order by dragging and dropping them onto the target zone. You can [see an example of this exercise in action here](https://campus-no-cache.datacamp.com/courses/26093/84524?ex=4).

## The anatomy of a parsons exercise

An ordering exercise is comprised of different components

*   Items (the items that get ordered)
*   An exercise title
*   An exercise description
*   Exercise instructions

  

![](https://t24474497.p.clickup-attachments.com/t24474497/d4c1fe56-82db-4d79-9f8c-5cd56a29e75e/image.png)

## Guidelines and best practices

*   The maximum number of ordering items that can be used in an exercise is 5.
*   You can have up to two indentations in the code pattern you're teaching
*   The exercise title cannot exceed 25 characters.
*   The exercise description needs to embed an element of storytelling ([see here](http://The exercise description needs to embed an element of storytelling (see here)))
*   Instructions need to be clear and in bullet points (even if one)
*   Hints need to be clear, but not give away the answer, and in bullet points (even if one)
