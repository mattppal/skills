## Lesson 1.3: "The Four P's Framework"

**Target:** 350-400 words, ~3.5-4 minutes  
**Objectives:** Master Four P's framework, understand systematic application

### Script Structure

**Opening (30s)**
- Bridge: "Method behind the magic"
- Four P's make vibe coding systematic

**Four P's Breakdown (180s)**

**Plan (45s):** Define problem before code
- Key questions: Who? What problem? Core functionality?
- Common mistake: Jumping to implementation

**Prompt (45s):** Communicate effectively with AI
- Best practices: Be specific, provide context, iterate
- Quality prompts = quality results

**Perfect (45s):** Refine through testing
- Your role: Quality controller, UX designer
- Good enough to ship vs. perfect

**Publish (45s):** Deploy and share
- Replit one-click deployment
- Getting feedback, measuring success

**Framework in Action (60s)**
- Larger project example
- Same framework, different scale

**Chapter Wrap (30s)**
- Accomplished: Mindset + experience + framework
- "You think like a vibe coder now"

1. "Method behind the magic"
2. Four P's framework flow
3-6. Each P with principles and examples
7. Framework applied to larger project
