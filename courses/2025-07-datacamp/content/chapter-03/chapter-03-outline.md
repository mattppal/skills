## Lesson 3.1: "Context Engineering vs. Prompt Engineering"

**Target:** 350-400 words, ~3.5-4 minutes  
**Objectives:** Master context engineering, understand AI conversation state management

### Script Structure

**Opening Hook (30s)**
- Bridge from Chapter 2: "You've got the foundation running"
- Problem: AI "forgets" context as projects grow
- Solution: Context engineering for sustained collaboration

**The Context Problem (60s)**
- What happens: AI loses track of project goals, decisions, architecture
- Why it matters: Inconsistent suggestions, repetitive explanations
- Real example: Adding features to feedback app without context

**Context Engineering Explained (120s)**

**Definition (40s):** Managing AI's understanding over time
- Not just better prompts—sustained project memory
- Think conversation state, not individual requests
- Your role: Context curator and conversation architect

**Key Techniques (40s):** Practical context management
- Project summary updates: "Here's where we are..."
- Decision documentation: "We chose X because..."
- Architecture reminders: "Our app structure is..."

**vs. Prompt Engineering (40s):** Different tools, different jobs
- Prompt engineering: Better individual requests
- Context engineering: Better ongoing collaboration
- When to use each approach

**Context in Action (75s)**
- Live demonstration: Managing AI context for feedback app
- Before/after: Same request with poor vs. rich context
- The difference in AI responses and suggestions
- Building context incrementally

**Best Practices (45s)**
- Start each session with context refresh
- Document key decisions as you make them
- Update AI on project evolution
- Context is investment, not overhead

**Bridge (15s)**
- "Now let's apply this to building real features"

### Slides
1. Hook: "AI needs memory too"
2. The context problem in growing projects
3. Context engineering definition
4. Key techniques: Summary, decisions, architecture
5-6. Live demo: Poor vs. rich context
7. Best practices framework
8. Transition: "Context to features"
